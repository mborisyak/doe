27 May 2026
=========================

Record schema (store-example/, STORAGE.md)
- Batch/design `experiments` is a dict {exp-N: {conditions, measurements}};
  dropped redundant `space` and `observable`; `x`->`conditions`,
  experiment-N -> exp-N
- measurements / predictions / expected are quantity-keyed dicts
  {timestamps, <obs>, sigma_<obs>?} -- `timestamps` is a key inside the dict.
  Real batch measurements carry no sigma; predictions/expected (model output)
  carry predictive std as sigma_<obs>
- predictions nested {batch: {exp: {...}}}; expected {model/v: {exp: {...}}}
- GP model `observables` is a list of names (ODE keeps a dict name->expr)

doe/dataset.py (new)
- assemble(paths) merges batch files by (filename-stem, exp); from_store(store,
  names); .series(obs) bridges to the inference core's generic shape; CLI
  `python -m doe.dataset a.json b.json -o out.json`

Scripts store-free; the MCP layer is the sole store authority
- mle.py / new_exp.py take file paths (--model spec, --batches/--history,
  --output RESULT.json), no store/locks/names; emit a plain result JSON
- mcp_engine materialises resolved inputs to a temp dir, runs the store-free
  script as a subprocess, reads result.json; compute_tools resolves refs,
  reserves+locks the output name, runs, then stamps + commits the record + a
  thin tool_result, releases. The MCP layer is the only writer.
- tool_result trimmed to {tool, status, inputs, references} (dropped the
  envelope blob that duplicated the result)

doe/store.py
- lock() returns a dual-use _Reservation: context manager AND acquire()/
  release() pair (like threading.Lock), with a guarded __del__ safety net;
  the old _lock contextmanager split into _reserve / _release

GP surrogate tools (doe/gp/surrogate.py, scripts/gp_*.py, mcp/*)
- New tools: create_gp, hyper_fit_gp, fit_gp, predict_gp, doe_gp,
  discriminatory_doe_gp. The GP maps (conditions, t) -> one observable; a single
  observable is enforced and outputs are keyed by observable name
- doe_gp = batch BALD (1/2 logdet(I + noise^-1 Sigma_BB)); discriminatory_doe_gp

  fitted_model's auxiliary
- Run as subprocess scripts via the engine, uniform with the ODE tools

doe/gp/kernels.py

=========================

doe/common/common.py
- ODE solver switched from Dopri5 to Tsit5
- Step size changed from 1e-2 to 0.1
- Added explicit ConstantStepSize() controller, max_steps, and DirectAdjoint()
  needed for compatibility with jacfwd through the ODE solve

doe/doe/fisher.py (complete rewrite)
- Takes an ODEModel directly (no neural network surrogate)
- Takes timestamps: jax.Array instead of t0/t1/n_timestamps (fixed measurement grid)
- Parameters passed to propose() in natural space and encoded internally
- Encoding/decoding of conditions and parameters via logit/sigmoid transforms
- Jacobian computed with jacfwd (forward-mode over reverse-mode) for
  compatibility with DirectAdjoint
- Line search extracted into ArmijoLineSearch; Fisher accepts one as
  line_search= or defaults to ArmijoLineSearch(x_min=-3, x_max=3)
- _propose uses jax.lax.scan with value_and_grad + line_search.update per iteration

doe/doe/line_search.py (new file)
- ArmijoLineSearch: backtracking line search following the optax
  GradientTransformation interface (init / update)
- init(params) returns an ArmijoState (currently stateless)
- update(grads, state, params, *, value_fn, value=None) runs the Armijo
  while-loop and returns (updates, new_state), compatible with optax.apply_updates
- Optional box projection via x_min / x_max

doe/doe/__init__.py
- Now exports Fisher, ArmijoLineSearch, ArmijoState
